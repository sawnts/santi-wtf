step by step

1. write in obsidian

i write and edit notes in obsidian, a note-taking app on my computer. obsidian stores everything as plain text files (markdown) in a folder on my computer.

the public garden lives in a folder called public. anything in there can potentially be published. anything starting with _ (like this _system folder) stays private.

2. run the sync

when i'm ready to publish, i run a command:

cd garden && npm run sync

this runs a script called build.js that:

  • reads all the markdown files in my obsidian public folder
  • converts them to html (web pages)
  • processes wikilinks ðŸ”’ into clickable links
  • figures out backlinks (what links to what)
  • generates an index of all notes with their metadata
  • skips anything starting with _

3. push to github

after the sync, i push the changes to github:

git add .
git commit -m "update garden"
git push

github is like a cloud backup for code. it stores all the files and tracks every change i've ever made.

4. github pages deploys

github pages automatically takes whatever is in the repository and makes it available on the internet. within a minute or two of pushing, the changes are live at santi.wtf.

what the build script creates

for each note, the script generates:

  • an html file with the content
  • metadata (title, stage, dates, tags)
  • a list of outgoing links
  • a list of backlinks (incoming links)

all this gets bundled into index.json, which the garden uses to render the folder tree, search, graph view, and backlinks section.

making changes

to add a new note:

  1. create a new .md file in obsidian
  2. add frontmatter (title, stage, planted date, tags)
  3. write the content
  4. run sync and push

to edit a note:

  1. edit in obsidian
  2. update the tended date in frontmatter
  3. run sync and push

to delete a note:

  1. delete or move the file out of public
  2. run sync and push
